Subiendo cambios

Si has estado trabajando por un timepo y notas que el repositorio del curso ha cambiado y deseas lo nuevo:

  • Mire el "Estado Actual" y asegurese de no haber modifiacdo nada

      git status

  • Suba los cambios

      git pull origin master

Haga un Fork ( ¡no clone! )

Para grabar una copia de su propias soluciones en nuestros propio repositorio de GitHub, tendra que hacer un "Fork" del curso en GitHub. Le proporcionara una nueva url para Git de sus cambios realizados, la cual podra clonar despues. Por ejemplo, si su usuario de GitHub es: ladyhaha, su comando de clonar es

    git clone git://github.com/ladyhaha/learn_ruby.git

Subir cambios dentro del nuevo "repo" es un poco "lioso" y probablemente deberia pedir ayuda antes de hacerlo, aunque aqui estan las instrucciones.

  • Añadir una referencia al repositorio remoto original

      git remote add ultrasaurus git://github.com/ultrasaurus/learn_ruby

  • Bajarse la rama "Master" del repositorio remoto. Esto crea una rama local llamada ultrasaurus/master

      git pull origin master

  • Ahora fusione el ultrasaurus/master en su master rama

      git merge ultrasaurus/master

  • Finalmente baje su propia rama master de nueavo a su rama origin del repositorio github remoto.

      git push origin master

Si necesita entender todas estos conjuros magicos de Git, vea este gran screencast: http://blip.tv/file/4094854 (mas en http://www.gitcasts.com)
